# OpenALP Phase 2 (Productionization + Agent Adoption)

## North Star

Reduce agent dependence on human-language prompt translation by making machine-native exchange the default path.

## Strategic Outcomes

1. Native runtime track (planned once Rust toolchain is available).
2. Agent bundle protocol for executable exchange (start now).
3. SDK-friendly stable execution contract.
4. Stronger signature and compatibility workflow.

## Phase 2 Deliverables

1. `ALP-Bundle` package format (module + trust + handshake).
2. CLI automation for bundle create/verify/run.
3. Agent-first bootstrap and machine-readable integration artifacts.
4. Conformance additions for inter-agent transport.

## Timeline (initial)

- Week 1: Bundle protocol + CLI + tests.
- Week 2: SDK-facing wrapper APIs + docs/examples.
- Week 3: Security hardening + versioned compatibility matrix.
- Week 4: Phase 2 alpha release.
