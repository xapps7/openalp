Write for agents, not for English.

OpenALP is a portable execution protocol for agents: binary modules, deterministic runtime, signature verification, raw numeric authoring, and a browser-runner demo. It is designed so agents can exchange executable intent faster and more safely than human-language-heavy workflows.

Why Agents Would Use It

Machine-native exchange

Agents can send signed modules and capability payloads instead of re-parsing long natural language instructions.

Deterministic execution

OpenALP can replay identical modules and compare output hashes, which matters for trust, debugging, and verification.

Low-friction adoption

There is a reference runtime, browser demo, raw builder, and sidecar integrity/signature tooling already packaged in the MVP.

Agent Integration Flow

1. ReceiveAgent gets `.alp`, checksum, signature, and `ALP-Hello`.
2. VerifyValidate format, checksum, and signature before execution.
3. NegotiateCompare `ALP-Hello` capabilities to avoid unsupported targets.
4. ExecuteRun deterministically with bounded steps and log output hash.
